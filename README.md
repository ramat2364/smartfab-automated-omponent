# SmartFab Automated Components — Digital Transformation Platform

A modern, containerized, multi-page industrial production scheduling, machine diagnostics, quality monitoring, and maintenance management platform built for **SmartFab Automated Components Pvt. Ltd.**

This system replaces manual Excel/paper operations across three manufacturing plants (**Pune, Nashik, Chennai**) with a unified digital interface tailored for specific operational roles (CEO, Plant Heads, production managers, quality checkers, and maintenance engineers).

---

## 🛠️ Tech Stack & Architecture

- **Frontend:** React + Next.js (App Router), TypeScript, Tailwind CSS, Recharts.
- **Backend:** Node.js + Express, TypeScript, REST API.
- **Database:** PostgreSQL with Prisma ORM (full relations for plants, lines, machines, readings, alerts, and logs).
- **Cache & Queues:** Redis + BullMQ (manages repeating cron jobs for parameter monitoring, specific energy aggregation, alert validation, and report preps).
- **Auth:** JWT with Refresh Token Rotation, Cookie/Header validations, and Role-Based Access Control (RBAC) scoping on every API.
- **AI Integrations:** Anthropic Claude API (with structured heuristic local fallback to run instantly without keys).
- **Deployment:** Dockerized architecture orchestrating Database, Redis, REST API, and Next.js.

## 🚀 Installation & Launch (One-Command Setup)

Ensure you have **Docker** and **Docker Compose** installed on your machine.

### Step 1: Start the services
From the project root folder, boot the docker containers:
```bash
docker-compose up --build -d
```
*Alternatively, if you have npm locally on the host, run: `npm run docker:up`*

### Step 2: Prepare Database & Seed Data
Once the database container is online (takes 5-8 seconds), run migrations and seed the 90 days of operational records:
```bash
# Apply Prisma Migrations
docker-compose exec backend npx prisma migrate dev --name init

# Seed the historical data
docker-compose exec backend npm run seed
```
*Alternatively, you can run: `npm run setup` to automatically launch, wait, migrate, and seed in one command!*

### Step 3: Access the platform
- **Frontend Panel:** Open `http://localhost:3000` in your web browser.
- **Backend API:** Access `http://localhost:5001/health` to verify server state.
- **Static Defect Photos:** Served dynamically at `http://localhost:5001/uploads/`.

## 📺 Platform Modules

### 1. Executive Dashboard (`/executive`)
Consolidated performance index comparing Chakan (Pune), Nashik, and Chennai side-by-side.
- **Cross-plant KPIs:** Live comparison of OEE averages, downtime hours, logged scrap units, and power costs.
- **90-Day Performance Trends:** Recharts line graphs showing daily plant OEE levels.
- **AI Executive Summary:** One-click weekly recap report generated by AI, identifying bottleneck locations and carbon saving certificate earnings.
- **Drill-down:** Clicking on any plant card filters the context and takes the CEO to that plant's production views.

### 2. Production Scheduling (`/production`)
- **Shift logging:** Supervisors manually record shift completions, total outputs, scrap count, and downtime minutes.
- **Production history:** Filter logs by line, date range, and location; export compiled records as CSV.
- **Line details:** 30-day Area charts plotting volumetric outputs vs. OEE trends, listing active machines.

### 3. Machine Health & Diagnostics (`/machines`)
- **Live machine metrics:** Tracks operating parameters (temperature, vibration speed, RPM) across machines.
- **Diagnostics charts:** Click into a machine to inspect 24-hour parameter trends with upper and lower operating limit reference lines.
- **AI Predictive Alerts:** Lists predictive alerts. Each includes an AI write-up on predicted failure modes (e.g. bearing wear), urgency level, and suggested actions.
- **Work Orders:** Assign alerts to technicians and submit resolution forms (downtime log, parts replaced) to clear alerts.

### 4. Quality Control (`/quality`)
- **Defect logging:** Register scrap items, select line/machine, attach photos, and log symptoms.
- **AI Root Cause Assist:** Consult the AI system during logging to analyze description details and suggest categories (Material, Calibration, Tooling wear).
- **RCA Analytics:** Charts displaying defect rates by Category, Part Numbers, and types.

### 5. Energy Management (`/energy`)
- **Overview:** Daily tariff costing compared against grid consumption (kWh).
- **Asset Breakdown:** Rankings of top-drawing machinery to schedule peak-tariff staging.
- **PAT Compliance:** Monitors regulatory BEE targets vs. actual specific energy consumption (SEC), calculating tradable Energy Savings Certificates.

## 🔄 Background Processing (BullMQ)

Diagnostic data and reporting jobs run on automated background schedules:
1. `parameter-monitoring-queue`: Triggers every 20 seconds to monitor operating parameters across registered machines.
2. `alert-evaluation-queue`: Triggers every 30 seconds to evaluate machine operating limits. If limits are exceeded, it generates a maintenance alert and dispatches AI diagnostic analysis.
3. `energy-aggregation-queue`: Triggers every 60 seconds to process power grid consumption data.
4. `report-generation-queue`: Compiles end-of-day summary reports.
